## 六、关键技术要点

### 6.1 下载链路

| 页面 | 工具 | 方式 |
|------|------|------|
| Product (案例二) | `downloadWithAxios` (`download-axios-clean.js`) | 纯净 axios 实例，`responseType: 'arraybuffer'`，无拦截器干扰 |
| Part (案例三) | `downloadByStream` (`download.js`) | 原生 fetch，手动携带 `Authorization` Token |

两种方式均正确处理：
- JSON 错误体识别
- `Content-Disposition` 文件名解析（RFC 5987）
- 二进制 Blob 保存

### 6.2 工作流状态机

| 状态 | 允许操作 | 禁止操作 |
|------|----------|----------|
| `CHECKED_IN` | 检出、删除 | 修改 |
| `CHECKED_OUT` | 修改、检入 | 删除 |

`wsCode()` 函数统一将后端多种格式（`INWORK`、`CHECKEDIN`、JSON 对象等）归一化为 `CHECKED_IN` / `CHECKED_OUT`。

### 6.3 附件三态（案例三 Part）

| 用户操作 | 前端设置 | 后端行为 |
|----------|----------|----------|
| 选择新文件 | `selectedRawFile` = 文件对象, `clearFile` = false | 替换原附件 |
| 手动移除文件 | `selectedRawFile` = null, `clearFile` = true | 清空附件 |
| 不动文件 | `selectedRawFile` = null, `clearFile` = false | 保持原附件 |
