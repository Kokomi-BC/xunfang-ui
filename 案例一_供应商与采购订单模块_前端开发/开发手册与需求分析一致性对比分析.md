## 三、需补充的关键内容

### 3.1 业务背景与目标

建议在手册"一、项目概述"中补充：

> **背景**：企业在采购协同中需要统一维护供应商主数据（编码、名称、联系人、联系方式、类型、供货范围与合作状态等），并支撑采购单等业务模块检索、选择与校验。
>
> **目标**：
> 1. 建设供应商管理模块，提供供应商的查询、分页、查看、创建、更新、删除（支持批量）等能力
> 2. 建设采购订单管理模块，提供采购订单的查询/分页/查看/新增/修改/删除与状态流转
> 3. 前端具备筛选查询 + 列表 + 编辑弹窗交互
> 4. 后端对接 iDME 平台完成数据落地

### 3.2 数据模型

#### 供应商（XfSupplier）

| 字段名 | 类型 | 约束/校验 |
|-------|------|----------|
| id | String | 主键，更新/删除必填 |
| supplierCode | String | 建议唯一，必填，≤32 |
| supplierName | String | 必填，≤64 |
| linkMan | String | 选填，≤32 |
| linkPhone | String | 选填，手机号/固话格式 |
| linkEmail | String | 选填，Email 格式 |
| supplierType | String | 1-原材料/2-设备/3-零件；必填 |
| address | String | 选填，≤128 |
| scopeOfSupply | String | 选填，≤128 |
| cooperativeStatus | String | 1-合作中/2-已暂停/3-已中止；必填 |
| remark | String | 选填，≤256 |

#### 采购订单（XfPurchaseOrder）

| 字段名 | 类型 | 约束/校验 |
|-------|------|----------|
| id | String | 主键，更新/删除必填 |
| purchaseOrderCode | String | 必填，建议唯一，≤32 |
| purchaseDate | Date | 必填（yyyy-MM-dd） |
| supplierName | String | 必填 |
| supplierId | String | 建议同时保存 |
| supplierLinkMan | String | 选填，随供应商带出 |
| materialCode | String | 必填 |
| materialName | String | 必填 |
| specificationsModels | String | 选填，≤64 |
| purchaseQuantity | Long | 必填，>0 |
| unit | String | 必填 |
| unitPrice | Float | 必填，≥0 |
| totalPrice | Float | = 数量×单价，两位小数 |
| status | String | 1-待确认/2-已确认/3-已发货/4-已完成 |

### 3.4 供应商 ID 联动保存

在采购订单表单和供应商选择逻辑中增加 `supplierId`：

**表单初始值：**
```javascript
form: {
  id: null, ...,
  supplierId: null,  // 新增
  supplierName: null,
  supplierLinkMan: null,
  ...
}
```

**确认选择供应商时：**
```javascript
function confirmSelectSupplier() {
  if (!curRow.value) return proxy.$modal.msgWarning("请选择一个供应商")
  form.value.supplierId = curRow.value.id        // 新增
  form.value.supplierName = curRow.value.supplierName
  form.value.supplierLinkMan = curRow.value.linkMan
  supplierOpen.value = false
}
```
